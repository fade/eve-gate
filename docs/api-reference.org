* API Reference

Complete reference for eve-gate's 195 generated ESI API functions and
supporting utilities.

[[file:index.org][Back to Index]] | [[file:quickstart.org][Quickstart]]

** Function Naming Convention

API functions follow the pattern: =HTTP-METHOD-PATH-SEGMENTS=

Examples:
- =get-status= - GET /status/
- =get-characters-character-id= - GET /characters/{character_id}/
- =get-characters-character-id-wallet= - GET /characters/{character_id}/wallet/
- =post-characters-affiliation= - POST /characters/affiliation/

** Common Parameters

All generated API functions accept:

| Parameter    | Type   | Description                                    |
|--------------+--------+------------------------------------------------|
| =client=       | object | API client (first positional argument)         |
| =:datasource=  | string | "tranquility" (default) or "singularity"       |
| =:token=       | string | OAuth token override (optional)                |

Endpoint-specific parameters are passed as keywords matching the path/query
parameter names, with underscores converted to hyphens.

** API Client Utilities

*** Creating Clients

#+begin_src lisp
;; Basic API client
(eve-gate.api:make-api-client &key base-url user-agent 
                                   connect-timeout read-timeout 
                                   max-retries token-manager token)

;; High-level client (includes caching, etc.)
(eve-gate:make-eve-client &key config client-id client-secret redirect-uri
                               enable-performance-monitoring)
#+end_src

*** Generic API Calls

#+begin_src lisp
;; Call by operation ID
(eve-gate.api:api-call client "get_characters_character_id" 
                       :character-id 95465499)

;; HTTP verb methods
(eve-gate.api:api-get client "/characters/95465499/" &key query-params token)
(eve-gate.api:api-post client "/characters/affiliation/" &key content token)
(eve-gate.api:api-put client path &key content token)
(eve-gate.api:api-delete client path &key token)
#+end_src

** Response Handling

*** Return Values

API functions return multiple values:

#+begin_src lisp
(multiple-value-bind (body response)
    (eve-gate.api:get-characters-character-id client :character-id 95465499)
  (format t "Name: ~A~%" (gethash "name" body))
  (format t "Status: ~A~%" (eve-gate.core:esi-response-status response))
  (format t "ETag: ~A~%" (eve-gate.core:esi-response-etag response)))
#+end_src

*** Response Object

#+begin_src lisp
;; Response accessors
(eve-gate.core:esi-response-status response)    ; HTTP status code
(eve-gate.core:esi-response-body response)      ; Parsed body
(eve-gate.core:esi-response-headers response)   ; Header alist
(eve-gate.core:esi-response-etag response)      ; ETag value
(eve-gate.core:esi-response-expires response)   ; Cache expiry
(eve-gate.core:esi-response-pages response)     ; Total pages (if paginated)
#+end_src

** Pagination

Many endpoints return paginated results:

#+begin_src lisp
;; Get first page
(multiple-value-bind (items response)
    (eve-gate.api:get-universe-types client :page 1)
  (let ((total-pages (eve-gate.core:esi-response-pages response)))
    (format t "Page 1 of ~A~%" total-pages)
    ;; Fetch remaining pages
    (loop for page from 2 to total-pages
          append (eve-gate.api:get-universe-types client :page page))))
#+end_src
