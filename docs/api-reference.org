* API Reference

Complete reference for eve-gate's 195 generated ESI API functions and
supporting utilities.

[[file:index.org][Back to Index]] | [[file:quickstart.org][Quickstart]]

** Function Naming Convention

API functions follow the pattern: =HTTP-METHOD-PATH-SEGMENTS=

Examples:
- =get-status= - GET /status/
- =get-characters-character-id= - GET /characters/{character_id}/
- =get-characters-character-id-wallet= - GET /characters/{character_id}/wallet/
- =post-characters-affiliation= - POST /characters/affiliation/

** Common Parameters

All generated API functions accept:

| Parameter    | Type   | Description                                    |
|--------------+--------+------------------------------------------------|
| =client=       | object | API client (first positional argument)         |
| =:datasource=  | string | "tranquility" (default) or "singularity"       |
| =:token=       | string | OAuth token override (optional)                |

Endpoint-specific parameters are passed as keywords matching the path/query
parameter names, with underscores converted to hyphens.

** API Client Utilities

*** Creating Clients

#+begin_src lisp
;; Basic API client
(eve-gate.api:make-api-client &key base-url user-agent
                                   connect-timeout read-timeout
                                   max-retries http-client
                                   token-manager token)
;; :http-client - optional pre-built http-client struct.  When supplied,
;;                the timeout / retry / user-agent keywords are ignored;

;;                timeout / retry / user-agent keywords.

                               enable-performance-monitoring)

** Request Scheduler

The high-level submission surface (see [[file:concurrency.org#request-scheduler][Concurrency — Request Scheduler]]
for an architectural overview). All symbols are in =eve-gate.concurrent=.

*** Construction and lifecycle

#+begin_src lisp
(make-request-scheduler &key cache-manager rate-limiter http-client engine
                             (budget-threshold 30) (budget-resume 50)
                             (jitter-margin 5)
                             (max-queue-depth 5000) (worker-count 8)
                             (rate-limit-timeout 30.0)
                             (bootstrap-queue-depth 80000)
                             (bootstrap-pacing 5.0)
                             (failure-ring-size 256)
                             (dispatcher-poll-interval 0.25))
;; Returns a request-scheduler. The rate-limiter, http-client, and engine
;; are built automatically when not supplied. Pass a cache-manager to
;; enable the cache short-circuit.

(start-scheduler scheduler)         ; idempotent; starts engine + dispatcher
(stop-scheduler  scheduler &key (wait t))  ; idempotent; joins dispatcher
(with-scheduler  (var &rest args) &body body)  ; unwind-safe start/stop
#+end_src

*** Submission verbs

#+begin_src lisp
;; Single refresh — priority is required.
(submit-refresh scheduler &key operation-id params priority deadline-seconds
                               batch-id on-complete on-skip on-fail)
;; Returns a refresh-handle.

;; Multi-spec batch — wraps each spec's callbacks for progress tracking.
(submit-refresh-batch scheduler specs
                      &key batch-id on-progress on-batch-complete)
;; SPECS is a list of plists, each accepted by submit-refresh's keyword
;; args (per-spec :batch-id is overridden by the outer :batch-id).
;; Returns a list of refresh-handles, in spec order.

;; Bulk drain — paced by the scheduler's bootstrap-pacing token bucket.
(submit-bootstrap scheduler specs
                  &key batch-id deadline-seconds on-progress on-complete)
;; SPECS is a list of plists with :operation-id, :params, :on-complete,
;; :on-skip, :on-fail (no :priority — bootstrap is implicit).
;; Returns a bootstrap-handle. Signals scheduler-queue-full if the batch
;; would push slot 4 past bootstrap-queue-depth.
#+end_src

Priority must be one of =:critical=, =:hot=, =:warm=, =:cold= on
=submit-refresh=. =:bootstrap= is rejected by =submit-refresh= and routed
through =submit-bootstrap= instead.

*** Cancellation

#+begin_src lisp
(cancel-refresh   scheduler handle)             ; T if cancelled, NIL if no-op
(cancel-bootstrap scheduler bootstrap-handle)   ; integer count cancelled
#+end_src

Only =:pending= handles are cancellable. Dispatched handles complete with
their eventual response and the consumer's normal callback fires.

*** Observation

#+begin_src lisp
(scheduler-state scheduler &key (recent-failures-limit 20))
;; Returns a plist with keys :budget :queue :bootstrap :engine :cache
;; :scheduler :recent-failures. Sub-blocks are NIL when the corresponding
;; subsystem is absent (no cache-manager, no active bootstrap, etc.).
;; Safe to call at sustained 1Hz.

(reset-scheduler-stats scheduler)
;; Zeros cumulative scheduler-private counters. Does not touch in-flight,
;; active handles, or active bootstrap batches.
#+end_src

*** =refresh-handle= accessors

| Accessor                       | Description                                  |
|--------------------------------+----------------------------------------------|
| =refresh-handle-id=            | gensym for logging                           |
| =refresh-handle-status=        | =:pending=, =:dispatched=, =:completed=, =:cancelled= |
| =refresh-handle-completion=    | the =refresh-completion= when settled        |
| =refresh-handle-operation-id=  | normalized operation-id string               |
| =refresh-handle-endpoint=      | resolved path with substitutions             |
| =refresh-handle-priority=      | consumer-facing keyword priority             |
| =refresh-handle-batch-id=      | pass-through batch-id, if any                |

Status and completion reads are atomic at the word level on SBCL — no
lock needed for snapshot reads.

*** =refresh-completion= accessors

| Accessor                                | Description                              |
|-----------------------------------------+------------------------------------------|
| =refresh-completion-outcome=            | =:ok=, =:cache-hit=, =:skipped=, =:failed= |
| =refresh-completion-event=              | =esi-completion-event= instance or NIL   |
| =refresh-completion-data=               | parsed response body (=:ok=, =:cache-hit=) |
| =refresh-completion-response=           | =esi-response= struct (=:ok= only)       |
| =refresh-completion-etag=               | ETag from response or cache              |
| =refresh-completion-cache-tier=         | =:l1=, =:l2=, or NIL                     |
| =refresh-completion-attempt-count=      | 1, or higher if 420-retried              |
| =refresh-completion-enqueued-at=        | universal-time at =submit-refresh=       |
| =refresh-completion-dispatched-at=      | universal-time at engine pickup, or NIL  |
| =refresh-completion-completed-at=       | universal-time at outcome                |
| =refresh-completion-wait-time=          | seconds in queue                         |
| =refresh-completion-rate-limit-wait=    | seconds in the token gate                |
| =refresh-completion-http-latency=       | seconds in HTTP I/O, or NIL              |
| =refresh-completion-endpoint=           | resolved endpoint                        |
| =refresh-completion-operation-id=       | normalized operation-id                  |
| =refresh-completion-batch-id=           | pass-through batch-id, if any            |

*** =bootstrap-handle= accessors

| Accessor                              | Description                              |
|---------------------------------------+------------------------------------------|
| =bootstrap-handle-id=                 | gensym for logging                       |
| =bootstrap-handle-batch-id=           | grouping string                          |
| =bootstrap-handle-total=              | spec count at submit time                |
| =bootstrap-handle-completed-count=    | settled count (any terminal outcome)     |
| =bootstrap-handle-ok-count=           | succeeded count (=:ok= + =:cache-hit=)   |
| =bootstrap-handle-failed-count=       | failed count                             |
| =bootstrap-handle-skipped-count=      | skipped count                            |
| =bootstrap-handle-cancelled-p=        | flag set by =cancel-bootstrap=           |
| =bootstrap-handle-refresh-handles=    | the per-spec refresh-handles, in order   |

Reads on the counters require the bootstrap-handle's lock for a coherent
snapshot — the wrapped per-spec callbacks bump under the same lock. The
counters are also exposed through =scheduler-state= for active batches.

** Response Handling

*** Return Values

API functions return multiple values:

#+begin_src lisp
(multiple-value-bind (body response)
    (eve-gate.api:get-characters-character-id client :character-id 95465499)
  (format t "Name: ~A~%" (gethash "name" body))
  (format t "Status: ~A~%" (eve-gate.core:esi-response-status response))
  (format t "ETag: ~A~%" (eve-gate.core:esi-response-etag response)))
#+end_src

*** Response Object

#+begin_src lisp
;; Response accessors
(eve-gate.core:esi-response-status response)    ; HTTP status code
(eve-gate.core:esi-response-body response)      ; Parsed body
(eve-gate.core:esi-response-headers response)   ; Header alist
(eve-gate.core:esi-response-etag response)      ; ETag value
(eve-gate.core:esi-response-expires response)   ; Cache expiry
(eve-gate.core:esi-response-pages response)     ; Total pages (if paginated)
#+end_src

** Pagination

Many endpoints return paginated results:

#+begin_src lisp
;; Get first page
(multiple-value-bind (items response)
    (eve-gate.api:get-universe-types client :page 1)
  (let ((total-pages (eve-gate.core:esi-response-pages response)))
    (format t "Page 1 of ~A~%" total-pages)
    ;; Fetch remaining pages
    (loop for page from 2 to total-pages
          append (eve-gate.api:get-universe-types client :page page))))
#+end_src
