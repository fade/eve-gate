#+TITLE: Eve-Gate
#+AUTHOR: Brian O'Reilly
#+OPTIONS: toc:t num:nil

A comprehensive EVE Online ESI (EVE Swagger Interface) API client library for
Common Lisp. Eve-gate provides auto-generated API bindings for all 195 ESI
endpoints, OAuth 2.0 authentication, multi-tier caching with ETag support,
rate limiting, and parallel request execution.

* Features

- *195 Generated API Functions*: Complete coverage of ESI endpoints across 20 categories
- *OAuth 2.0 Authentication*: Browser-based EVE SSO login with automatic token refresh
- *Multi-tier Caching*: Memory + file-based persistence with ETag revalidation
- *Rate Limiting*: Token bucket algorithm respecting ESI's floating window limits
- *Parallel Execution*: Concurrent request engine with bulk operations support
- *Type Safety*: Comprehensive type system for EVE Online entity IDs
- *Structured Logging*: JSON and text formatters with request correlation
- *Health Monitoring*: System health checks, SLA monitoring, and alerting

* Quickstart

** 1. Install the library

#+begin_src sh
cd ~/common-lisp/  # Or your ASDF source registry path
git clone https://github.com/fade/eve-gate.git
#+end_src

** 2. Load and create a client

#+begin_src lisp
(ql:quickload :eve-gate)
(in-package :eve-gate)

;; For public (unauthenticated) endpoints
(defvar *client* (make-eve-client))

;; For authenticated endpoints, you need OAuth credentials
(defvar *auth-client* 
  (make-eve-client :client-id "your-client-id"
                   :client-secret "your-client-secret"
                   :redirect-uri "http://localhost:29170/oauth/callback"))
#+end_src

** 3. Make API calls

#+begin_src lisp
;; Public endpoint: get server status
(eve-gate.api:get-status *client*)
;; => #<HASH-TABLE :COUNT 4>
;;    "players" -> 31234
;;    "server_version" -> "2162163"
;;    "start_time" -> "2026-04-13T11:00:00Z"
;;    "vip" -> NIL

;; Public endpoint: get character info
(eve-gate.api:get-characters-character-id *client* :character-id 95465499)
;; => #<HASH-TABLE> with name, corporation_id, birthday, etc.

;; Get multiple characters in parallel
(eve-gate.api:get-alliances *client*)
;; => #(99010079 99010193 99010194 ...)  ; Vector of alliance IDs

;; Paginated endpoint with type info
(eve-gate.api:get-universe-types *client* :page 1)
;; => #(0 1 2 3 ...) ; page of type IDs
#+end_src

** 4. Authenticate for private endpoints

#+begin_src lisp
;; Generate authorization URL and open in browser
(let ((url (eve-gate.auth:get-authorization-url 
            (eve-client-auth-client *auth-client*)
            :scopes '("esi-wallet.read_character_wallet.v1"
                      "esi-skills.read_skills.v1"))))
  (format t "Visit: ~A~%" url))

;; After user authorizes, exchange the code for tokens
(let ((auth-code "code-from-callback-url"))
  (authenticate-client *auth-client* auth-code))

;; Now authenticated endpoints work
(eve-gate.api:get-characters-character-id-wallet *auth-client* 
                                                  :character-id 95465499)
#+end_src

* API Reference

Eve-gate organizes functionality into several packages:

** eve-gate (Main Package)

High-level client interface.

| Symbol              | Type     | Description                          |
|---------------------+----------+--------------------------------------|
| =make-eve-client=     | function | Create configured EVE client         |
| =eve-client=          | class    | Main client object                   |
| =with-client=         | macro    | Execute with temporary client        |
| =authenticate-client= | function | Authenticate with OAuth code         |
| =configure-client=    | function | Runtime configuration updates        |

** eve-gate.api (API Functions)

195 generated ESI endpoint functions plus utilities.

#+begin_src lisp
;; Pattern: HTTP-METHOD-PATH-SEGMENTS
(get-characters-character-id client :character-id 12345)
(get-characters-character-id-wallet client :character-id 12345)
(post-characters-affiliation client :characters '(123 456 789))

;; All functions accept:
;;   CLIENT - api-client or http-client
;;   Keyword args matching endpoint parameters
;;   :token - Optional override OAuth token
;;   :datasource - "tranquility" (default) or "singularity"
#+end_src

Key utilities:

| Symbol            | Type     | Description                              |
|-------------------+----------+------------------------------------------|
| =make-api-client=   | function | Create API client                        |
| =api-call=          | function | Generic call by operation-id             |
| =api-get/post/put/delete= | function | HTTP verb convenience methods      |

* Configuration

Eve-gate supports multiple configuration sources with precedence:

1. Environment variables (highest)
2. JSON/Lisp config files  
3. Programmatic defaults (lowest)

** Environment Variables

| Variable                | Description                              |
|-------------------------+------------------------------------------|
| =EVE_GATE_CLIENT_ID=      | OAuth client ID                          |
| =EVE_GATE_CLIENT_SECRET=  | OAuth client secret                      |
| =EVE_GATE_LOG_LEVEL=      | Log level (trace/debug/info/warn/error) |
| =EVE_GATE_CACHE_ENABLED=  | Enable caching (true/false)              |
| =EVE_GATE_DATASOURCE=     | ESI datasource (tranquility/singularity) |

** Configuration File

#+begin_src lisp
;; ~/.config/eve-gate/config.lisp
(:http-timeout 30
 :http-retries 3
 :cache-enabled t
 :cache-default-ttl 300
 :log-level :info
 :worker-threads 4)
#+end_src

** Programmatic Configuration

#+begin_src lisp
(eve-gate.utils:initialize-config-manager 
 :environment :development
 :sources (list (eve-gate.utils:make-env-source)
                (eve-gate.utils:make-file-source "~/.config/eve-gate/config.lisp")))
#+end_src

* Error Handling

Eve-gate uses the Common Lisp condition system for errors:

#+begin_src lisp
(handler-case 
    (eve-gate.api:get-characters-character-id *client* :character-id 0)
  (eve-gate.core:esi-not-found (c)
    (format t "Character not found: ~A~%" (esi-error-message c)))
  (eve-gate.core:esi-rate-limit-exceeded (c)
    (format t "Rate limited, retry after ~A seconds~%" 
            (esi-rate-limit-retry-after c)))
  (eve-gate.core:esi-error (c)
    (format t "ESI error ~A: ~A~%" 
            (esi-error-status-code c)
            (esi-error-message c))))
#+end_src

Condition hierarchy:

- =esi-condition= (base)
  - =esi-deprecation-warning= (warning)
  - =esi-error= (error base)
    - =esi-client-error= (4xx)
      - =esi-bad-request= (400)
      - =esi-unauthorized= (401)
      - =esi-forbidden= (403)
      - =esi-not-found= (404)
      - =esi-rate-limit-exceeded= (420)
    - =esi-server-error= (5xx)
      - =esi-internal-error= (500)
      - =esi-bad-gateway= (502)
      - =esi-service-unavailable= (503)
      - =esi-gateway-timeout= (504)
    - =esi-network-error=
      - =esi-connection-timeout=
      - =esi-read-timeout=

* Performance Tips

1. *Enable caching*: Reduces API calls significantly for repeated data
2. *Use bulk operations*: =bulk-get= executes requests in parallel
3. *Respect rate limits*: Eve-gate handles this automatically, but be mindful
4. *Reuse clients*: Create one client and reuse it (thread-safe)
5. *Monitor health*: Use =(quick-health)= to check system state

#+begin_src lisp
;; Bulk fetch multiple characters efficiently
(eve-gate.concurrent:bulk-get *client* 
  '("/characters/12345/" "/characters/67890/" "/characters/11111/"))

;; Check cache effectiveness
(eve-gate.cache:cache-summary (eve-client-cache-manager *client*))
#+end_src
